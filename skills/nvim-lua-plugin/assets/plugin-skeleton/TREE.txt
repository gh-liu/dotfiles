plugin-skeleton/
├── plugin/
│   └── myplugin.lua           -- 启动时执行：只注册 commands/keymaps/autocmd
├── lua/
│   └── myplugin/
│       ├── init.lua           -- 模块入口：导出公共 API
│       ├── config.lua         -- 默认配置 + setup(opts)
│       ├── actions.lua        -- 具体功能实现
│       └── health.lua         -- :checkhealth 支持
└── doc/
    └── myplugin.txt           -- vimdoc 文档

使用方法：
1. 复制整个目录结构到你的插件仓库根目录
2. 全局替换 'myplugin' 为你的插件名
3. 修改各文件中的示例代码为你的实际功能
4. 运行 :helptags <doc-directory-path> 生成 vimdoc tags

关键原则（见 :h lua-plugin）：
- plugin/*.lua 顶层避免 require() 重模块；把 require() 移到回调里
- setup() 只 merge 配置，不做昂贵初始化
- 优先导出 <Plug> 映射，而非自动创建全局 keymap
